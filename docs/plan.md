# `coding-memory` — Refined MVP Implementation Plan

## Context

The spec at `docs/project.md` defines a local-first TypeScript CLI + MCP server that stores AI coding-session reasoning in SQLite outside the git repo and exposes it back to Claude Code. The project directory is currently empty (spec-only). This plan fixes six issues identified in the spec and applies six architectural refinements before implementation begins.

---

## Fixes Applied vs. Spec

| #   | Issue                                        | Resolution                                                                                                                                 |
| --- | -------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| 1   | `--manual` flag undefined                    | Deferred to v0.2. MVP supports `--summary-file` only.                                                                                      |
| 2   | `show` command missing from milestones       | Added to Milestone 5 alongside `search`.                                                                                                   |
| 3   | `Task:` heading missing from markdown parser | Added as an optional heading in `parseSummary`.                                                                                            |
| 4   | `relevanceReason` generation undefined       | Generated by `explainRelevance(ScoreBreakdown)` — formats non-zero score components.                                                       |
| 5   | Missing PKs on junction tables               | Added `UNIQUE(memory_id, value)` to `memory_alternatives`, `memory_risks`, `memory_followups`.                                             |
| 6   | Repo ID unstable for local-only repos        | Store both `repo_id` (hash) and `repo_path` (absolute). Search falls back to `repo_path` substring match if `repo_id` yields zero results. |

## Architecture

### Tech Stack (confirmed)

- TypeScript 5.x + Node 18+, **ESM** (`"type": "module"`, `module: "NodeNext"`)
- `commander` — CLI
- `better-sqlite3` — synchronous SQLite (no async in DB layer)
- `zod` — MCP input validation
- `@modelcontextprotocol/sdk` — stdio MCP server
- `vitest` — tests (in-memory SQLite per test file)
- `tsx` — dev runner, `tsc` — production build

### Key types (`src/memory/types.ts`)

```typescript
export type CodingMemory = {
  id;
  createdAt;
  updatedAt;
  repoId;
  repoPath?;
  repoName?;
  branch?;
  commitSha?;
  files;
  tags;
  task?;
  intent;
  summary;
  decision;
  why;
  alternativesRejected;
  risks;
  followUps;
  source;
  rawTranscriptPath?;
  embeddingText;
};

export type ScoreBreakdown = {
  total;
  sameRepo;
  fileOverlap;
  tagMatch;
  decisionMatch;
  whyMatch;
  summaryMatch;
  filePathMatch;
  recency;
};

export type SearchResult = { memory: CodingMemory; score: ScoreBreakdown; relevanceReason: string };
export type RepoContext = { repoRoot; repoId; repoPath; repoName; branch; commitSha; changedFiles };
```

## Milestone Plan

### Milestone 1 — Project Setup

**Files:** `package.json`, `tsconfig.json`, `.eslintrc.json`, `.prettierrc.json`, `src/config.ts`, `src/cli.ts` (skeleton), `src/commands/init.ts`, `tests/init.test.ts`

Key `package.json` settings:

- `"type": "module"`, `"bin": { "coding-memory": "./dist/cli.js" }`
- `"scripts": { "build": "tsc", "dev": "tsx src/cli.ts", "test": "vitest run" }`

`tsconfig.json`: `"module": "NodeNext"`, `"moduleResolution": "NodeNext"`, `"outDir": "dist"`, `"strict": true`

`src/config.ts`: exports `DATA_DIR = path.join(os.homedir(), '.coding-memory')`, `DB_PATH`, `CONFIG_PATH`, `resolveConfig()` (returns defaults if not initialized — never throws), `isInitialized()`.

`src/commands/init.ts`: accepts optional `{ dataDir? }` for testability. Creates dir, writes `config.json` (resolved absolute path, not `~`), opens DB, runs migrations.

**Acceptance:** `npm test` passes, `npm run build` produces `dist/cli.js`, `coding-memory init` creates `~/.coding-memory/`

---

### Milestone 2 — Database

**Files:** `src/db/connection.ts`, `src/db/schema.ts`, `src/db/migrations.ts`

`connection.ts`: `openDb(path)` — `new Database(path)`, sets `journal_mode = WAL` and `foreign_keys = ON`.

`schema.ts`: `MIGRATIONS: Migration[]` array. Migration v1 contains all `CREATE TABLE` + `CREATE INDEX` SQL. **Fixes:**

- `memory_alternatives`, `memory_risks`, `memory_followups` get `UNIQUE(memory_id, value)`
- Add `idx_memories_repo_path` index for fallback search
- `memory_files` / `memory_tags` keep `PRIMARY KEY (memory_id, file_path|tag)`

`migrations.ts`: Creates `migrations` table first, then applies pending versions using `db.exec()` (multi-statement safe).

**Acceptance:** DB has all tables, `migrations` table shows v1 applied, second `init` is idempotent.

---

### Milestone 3 — Git Helpers

**Files:** `src/git/git.ts`, `src/git/repoId.ts`, `src/git/diff.ts`, `src/memory/repoContext.ts`, `tests/gitHelpers.test.ts`

`git.ts`: All functions use `execSync({ cwd, encoding: 'utf-8' })`. Return `null` on failure (except `resolveCommit` which throws — invalid ref is a hard error).

`repoId.ts`: `getRepoId()` — normalize remote URL (strip `.git`, convert `git@` to `https://`), hash with `sha256`, truncate to 32 chars. Fall back to hash of `path.resolve(repoRoot)` if no remote.

`diff.ts`: `getChangedFilesForCommit()` — uses `git diff-tree --no-commit-id -r --name-only`.

`repoContext.ts`: `getRepoContext(cwd, commitRef)` — throws if not in a git repo. Returns full `RepoContext`.

Tests: `vi.mock('child_process')` to mock `execSync`. Test URL normalization, ID stability, and fallback behavior.

**Acceptance:** All git helper tests pass. Save command resolves metadata in a real repo.

---

### Milestone 4 — Save Command

**Files:** `src/memory/types.ts`, `src/memory/parseSummary.ts`, `src/memory/redactSecrets.ts`, `src/memory/saveMemory.ts`, `src/commands/save.ts`, `tests/redactSecrets.test.ts`, `tests/parseSummary.test.ts`, `tests/saveMemory.test.ts`

`parseSummary.ts`: Split on heading lines (`/^[A-Za-z][A-Za-z\- ]+:\s*$/m`). Parse 8 headings: `Intent`, `Summary`, `Decision`, `Why`, `Alternatives rejected`, `Risks`, `Follow-ups`, `Tags`, **plus `Task`** (Issue 3 fix). Lists parsed as bullet items (strip `- ` or `* `). Safe defaults for missing fields.

`redactSecrets.ts`: `SECRET_PATTERNS: Array<{name, pattern}>` covering GitHub tokens (`ghp_`, `gho_`), npm tokens (`npm_`), AWS keys (`AKIA...`), Bearer tokens, `.env`-style secrets (`KEY=value`), PEM blocks. `redactSecrets(input)` replaces all with `[REDACTED_SECRET]`.

`saveMemory.ts`: `generateMemoryId()` → `mem_${Date.now().toString(36)}${randomUUID().slice(0,8)}`. `buildEmbeddingText()` concatenates all content fields. `saveMemory(db, memory)` uses a `db.transaction()` wrapper; inserts to main table then all junction tables using `INSERT OR IGNORE`.

`save.ts`: Reads `--summary-file`, calls `getRepoContext()`, `parseSummary()`, `redactSecrets()` on all text fields, assembles `CodingMemory`, calls `saveMemory()`. Prints ID, repo, commit (8 chars), file count.

Tests: One test per secret pattern. `parseSummary` tests cover all 8 headings, missing fields, and bullet variants. `saveMemory` uses in-memory SQLite (`new Database(":memory:")`).

**Acceptance:** `coding-memory save --commit HEAD --summary-file memory.md` saves and prints ID.

---

### Milestone 5 — Search + Show Commands

**Files:** `src/memory/searchMemory.ts`, `src/output/format.ts`, `src/commands/search.ts`, `src/commands/show.ts`, `tests/searchMemory.test.ts`

`searchMemory.ts`: `scoreMemory(memory, query, contextRepoId, contextFiles): ScoreBreakdown` — exact scoring weights from spec (+10 same repo, +8 file overlap, +5 tag, +4 decision, +3 why, +2 summary, +2 file path, +1 recency ≤30 days). `explainRelevance(scores)` filters non-zero fields → `"Matched: same repo (+10), tag match (+5)"` (Issue 4 fix). `searchMemory()` queries DB, hydrates junction data, scores, sorts, limits. Repo fallback: if `repo_id` returns 0 results and `repoPath` available, retry with `repo_path LIKE '%...'`.

`format.ts`: `formatMemory(memory, verbose=false)` — concise output with id, commit, files, intent, decision, why, risks. `formatSearchResult(result)` appends `[relevanceReason]`.

`show.ts`: `runShow(id, {commit?})` — fetch by ID or commit SHA, print with `verbose=true`.

Tests: In-memory SQLite, 4-5 varied memories. Verify score ordering, `--limit`, `explainRelevance` output.

**Acceptance:** `coding-memory search "query"` returns ranked results; `coding-memory show <id>` shows verbose output.

### Milestone 6 — MCP Server

**Files:** `src/mcp/tools.ts`, `src/mcp/server.ts`, `src/commands/mcp.ts`

`tools.ts`: Zod schemas for all 4 tool inputs — `SearchMemoryInput`, `SaveMemoryInput`, `GetCommitMemoryInput`, `GetFileMemoriesInput` (matching spec section 14 exactly).

`server.ts`: `createMcpServer()` — `new Server(...)`, `ListToolsRequestSchema` handler returns 4 tool descriptors with JSON schemas from Zod. `CallToolRequestSchema` handler opens DB, routes to correct function, closes DB in `finally`. **MCP `save_coding_memory` also runs `redactSecrets()` on all text inputs.** `relevanceReason` in search output comes from `explainRelevance()`. Logs go to `process.stderr` only — stdout is reserved for JSON-RPC.

`mcp.ts`: `runMcp()` — checks `isInitialized()`, then `await createMcpServer()`.

**Acceptance:** `coding-memory mcp` starts; Claude Code can call all 4 tools and receive valid JSON.

## Critical Implementation Notes

1. **ESM imports**: Every `import` in `src/` must use `.js` extension (e.g. `import { foo } from './bar.js'`), even for `.ts` source files. This is enforced by `moduleResolution: NodeNext` and is the #1 gotcha.

2. **No async in DB layer**: `better-sqlite3` is synchronous. Use `db.transaction()` for multi-statement writes. MCP async handlers call sync DB functions — this is intentional.

3. **Migration SQL**: Use `db.exec()` for multi-statement migration SQL blocks. `db.prepare().run()` only supports single statements.

4. **Test isolation**: Every test file that writes to SQLite must use `new Database(":memory:")`. Never share DB state across test files.

5. **Config stores resolved paths**: `config.json` writes `os.homedir() + '/.coding-memory/memory.db'` as an absolute string, never the literal `~`.

6. **`--manual` flag**: Not implemented in MVP. The `post-commit.sample.sh` can mention it as a future command without wiring it to a handler.
